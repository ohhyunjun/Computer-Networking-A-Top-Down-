1. 인터넷 개요
   - 인터넷은 전 세계의 독립된 컴퓨터와 네트워크가 연결된 거대한 네트워크 시스템.
   - 독립된 컴퓨터나 기기를 연결해 통신할 수 있도록 하는 시스템.
   - 정보 교환 및 데이터 공유를 위해 다양한 프로토콜과 기술을 활용.

2. OSI(Open Systems Interconnection) 모델
   - 네트워크 통신 과정을 7개 계층으로 나눈 모델
     = 물리 계층: 데이터 전송을 위한 하드웨어적 신호 전달.
     = 데이터 링크 계층: 오류 제어, 프레임 동기화, MAC 주소 관리.
     = 네트워크 계층: IP 주소를 통한 패킷 라우팅 및 전달.
     = 전송 계층: TCP/UDP를 사용한 신뢰성 있는 데이터 전송.
     = 세션 계층: 송수신 간 세션 설정, 유지 및 종료.
     = 표현 계층: 데이터 변환, 압축, 암호화.
     = 응용 계층: 사용자와 네트워크 간 인터페이스 제공.

3. TCP/IP 구조
   - 인터넷에서 데이터를 주고 받기 위한 핵심 프로토콜.
     = 물리 계층: 전송 매체, 신호 인코딩 방식 정의.---비트
     = 데이터 링크 계층: ‘프레임’ 송수신 담당.---프레임
     = 네트워크 계층: IP주소 관리 및 패킷 라우팅(ABP, ICM 포함).---패킷
     = 전송 계층: TCP(신뢰성 보장), UDP(빠른 전송).---세그먼트<TCP>,데이터 그램<UDP>
     = 응용 계층: HTT, FTP, DNS, 이메일 등 네트워크 서비스 제공---메시지

4. 네트워크 장비
   - 리피터: 신호 증폭(물리 계층)
   - 허브: 여러 네트워크 장비와 장비를 연결하는 단순한 네트워크 장치(물리 계층), 단순히 입력된 데이터 모든 포트로 전달.
   - 브리지: MAC 주소 기반으로 두 개의 네트워크를 연결하고 분할하며 데이터 트래픽 관리(데이터 링크 계층), 데이터를 허브와 달리 필요한 포트로만 전달
   - 스위치: 네트워크에서 MAC/IP 주소를 기반으로 데이터를 효율적으로 전달(데이터 링크 계층), 허브와 달리 목적지로만 전송, 여러 장치가 동시 통신 가능
   - 라우터: 최적 경로를 찾아 데이터 패킷을 전달(네트워크 계층)
   - 게이트웨이: 서로 다른 네트워크 간 프로토콜 변환
